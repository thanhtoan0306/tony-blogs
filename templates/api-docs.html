Upload bài — POST /api/articles

Yêu cầu xác thực: header X-Upload-Password (mật khẩu upload) hoặc cookie upload_auth sau khi đăng nhập tại /upload.

Content-Type: application/json. Giới hạn body: 1 MB.

curl -sS -X POST {{.BaseURL}}/api/articles \
  -H "Content-Type: application/json" \
  -H "X-Upload-Password: YOUR_PASSWORD" \
  -d '{
    "title": "Tiêu đề bài viết",
    "slug": "tieu-de-bai-viet",
    "summary": "Mô tả ngắn 1–2 câu hiển thị trên thẻ bài.",
    "bodyHtml": "<p>Đoạn mở đầu.</p><h2>Mục chính</h2><p>Nội dung chi tiết.</p>",
    "category": "Việt Nam",
    "author": "Tony Blogs",
    "publishedAt": "2026-06-12T08:00:00Z",
    "tags": ["vietnam", "crypto"],
    "thumbnail": "https://images.unsplash.com/photo-1611974789855-9c2a0a7236a3?auto=format&fit=crop&w=1200&q=80",
    "visible": true
  }'

Phản hồi thành công (201):

{
  "ok": true,
  "slug": "tieu-de-bai-viet",
  "url": "/news/tieu-de-bai-viet",
  "title": "Tiêu đề bài viết"
}

Lỗi thường gặp:

  • 401 — thiếu hoặc sai mật khẩu
  • 400 — thiếu title, summary, bodyHtml hoặc slug trùng
  • 503 — server đang chạy chế độ JSON read-only (cần Firestore)

Schema JSON

FieldBắt buộcMô tả
title✓Tiêu đề hiển thị. Tiếng Việt có dấu được.
summary✓Tóm tắt 1–3 câu (~120–220 ký tự), dùng trên thẻ bài và SEO.
bodyHtml✓Nội dung HTML an toàn (xem mục HTML bên dưới).
slugURL path: chỉ a-z, 0-9, dấu -. Bỏ trống → tự sinh từ title.
categoryMặc định General. Ví dụ: Việt Nam, Crypto.
authorMặc định Crypto Desk. Nên dùng Tony Blogs cho bài hệ thống.
publishedAtISO 8601 UTC, ví dụ 2026-06-12T08:00:00Z. Bỏ trống → thời điểm upload.
tagsMảng chuỗi, ví dụ ["vietnam","btc"].
thumbnailURL ảnh HTTPS. Bỏ trống → /static/default-fallback-image.png.
visibletrue hiện trang chủ, false ẩn. Mặc định true.

Quy tắc viết bài (cho AI model)

  1. Ngôn ngữ: Viết tiếng Việt tự nhiên, rõ ràng, trừ khi chủ đề yêu cầu tiếng Anh.
  2. Cấu trúc: Mở bài 1 đoạn ngắn → <h2> cho từng mục → kết bằng tóm tắt hoặc <blockquote> disclaimer nếu là tin pháp lý/tài chính.
  3. Summary: Không lặp nguyên tiêu đề; nêu ai / cái gì / khi nào / vì sao quan trọng.
  4. Slug: Tiếng Việt không dấu, nối bằng -, ngắn gọn, mô tả chủ đề. Ví dụ: timeline-tai-san-dien-tu-vn-2026.
  5. Độ dài: Bài tin: 400–800 từ. Bài phân tích/timeline: 800–1500 từ. Tránh filler.
  6. Sự thật: Không bịa số liệu, ngày tháng, trích dẫn cơ quan. Nếu không chắc, ghi “theo báo cáo ngành” hoặc disclaimer.
  7. Pháp lý / tài chính: Thêm blockquote nhắc đây là tổng hợp, không thay văn bản pháp luật.
  8. Tags: 3–6 tag chữ thường, không dấu: chủ đề + địa lý + từ khóa (vd. vietnam, crypto, nq-05).
  9. Thumbnail: URL ảnh minh họa phù hợp chủ đề (Unsplash hoặc ảnh static). Tỷ lệ ngang, ≥ 800px rộng.
  10. Trùng slug: Kiểm tra trước khi upload; nếu trùng, đổi slug hoặc cập nhật thủ công qua Firestore.

HTML trong bodyHtml

Chỉ dùng thẻ sau (không script, iframe, style inline phức tạp):

  • <p>, <h2>, <h3>
  • <ul> / <ol> + <li>
  • <strong>, <em>, <a href="..."> (link ngoài HTTPS)
  • <blockquote>

Timeline / tin theo mốc thời gian: dùng <ul> với <strong>DD/MM/YYYY</strong> đầu dòng.

Ví dụ prompt cho AI

Bạn là biên tập viên Tony Blogs. Viết 1 bài tiếng Việt về [CHỦ ĐỀ].
Trả về JSON đúng schema POST /api/articles:
- title, slug (không dấu), summary (≤ 220 ký tự)
- bodyHtml: HTML hợp lệ, có h2, danh sách nếu cần
- category, author "Tony Blogs", tags[], thumbnail URL
- publishedAt ISO UTC, visible true
Sau đó gọi: curl -X POST {{.BaseURL}}/api/articles -H "X-Upload-Password: ..." -d @article.json
